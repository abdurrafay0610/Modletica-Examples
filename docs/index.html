1. Purpose

The purpose of this document is to provide a clear and comprehensive guide on how to use the Modletica tool to generate an UPPAAL model from C code.
Modletica is a custom built tool for the automated model extraction from C code.

2. Scope

This document will discuss step-by-step on how to generate an UPPAAL model from C code using the tool Modletica.
This document will also discuss some pre-processing that is required before providing the files to the tool. (Discussed further in the WI)

This document will not discuss the internal workings of Modletica. However, a reference to the Github: Modletica-Example will be given if any are interested in it. This github link contains detailed documentation and examples on how modletica converts C concepts to UPPAAL. ref 3.2.1

5. Definitions

5.1. Modletica: A custom built C to UPPAAL model extractor.
5.2. user_defined_datatypes: This is a file that contains common user defined data types. ref 3.1.3

This is like a helper file that we provide to our code for smooth C to model conversion.
This file is as-it-is copy pasted to UPPAAL, so if you are aware of the UPPAAL syntax, you can modify the file.
5.3. Defined Functions: This term refers to functions who were called and whose definition were found in the code provided by the user.
5.4. Undefined Functions: This term refers to functions who were called but their definition were not found in the code provided by the user.
5.5. Undefined Variables: This term refers to variables who are being used, but their declaration were not found in the code provided by the user.

7. Assumptions

7.1. Assumptions to run (modletica the tool):
7.1.1. JDK 19 (or above) version is installed on your system. ref 3.1.1
7.1.2. GCC/G++ Compilers should be accessible from the terminal. ref 3.1.4
7.1.3. The operating system should be Windows.
7.2. Assumptions for the code (from which we want to extract model from):
7.2.1. The code should be written in C language.
7.2.2. The code should be error free (it should not have any syntax errors).
7.2.3. The code should be in a compilable form.
7.2.4. There should be no variable shadowing in the code.
7.2.5. There is no recursion used in the code.

If any of these assumptions are not met, than there is no guarantee that the output model will be correct.

8. Procedure

8.0. Following preprocessing must be done (to expand macros) before providing the files to modletica:
8.0.1. Open your powershell or terminal.
8.0.2. Run the command:

gcc --version

It should show the gcc version. If not then install gcc installer. ref 3.1.4




8.0.3. For each of your files run the command (Replace "filename" with your actual filename):

gcc -E filename.c > filename_solved.c



8.0.4. Your filename_solved.c file now should have all of your libraries and macros preprocessed.
8.0.5. You cannot pass the file "filename_solved.c" as it is to the tool. You need to do the following:
8.0.5.1. The processed file will contain both expanded libraries and macros.
8.0.5.2. Create a new file (this is compulsory).
8.0.5.3. Copy the processed file content (except the expanded libraries) to the new created file.
8.0.5.4. When prompted in the preceding steps, provide these .c/.h files to the tool.

8.1 Run the Modletica tool.
8.1.1.1. Download the latest runnable jar from the ref 3.1.2
8.1.1.2. Double click on the downloaded jar file. It should start the Modletica tool.
8.2. Following working directories must be selected before proceeding:
8.2.1. Choose a log directory. All the logs created by the tool (for debugging purposes) will be created/stored in this directory.



8.2.2. Choose an xml directory. The final created xml (the UPPAAL model) will be saved/created/stored in this directory.



8.2.3. Provide the tool with a *Predefined data type* file. You can either create your own file or provide the tool with the already made file. ref 3.1.3




8.2.4. Press the continue button to go to the next screen which is "Files Management Page".



8.3. On the "Files Management Page" take the following steps:
8.3.1. At this point you need to provide your C files to the tool. The tool will extract the model from these files. Be sure to provide all the files that are relevant to the code that you want to extract a model from. You can provide the tool with C files in 2 ways:
8.3.1.1. By clicking the "Add C File" option and give it the C files one-by-one.



8.3.1.2. By clicking on the “Add Directory” option and provide it with the directory that contain all the relevant source files. The tool will automatically extract all .c and .h files.



8.3.2. If you want to remove any file:
8.3.2.1. Select the file from the drop down. And press the “Remove” option to remove it.



8.3.2.2. Alternately, you can remove all the files using the "Remove All" option.



8.3.3. If you want to make any changes in the previous page, press the "Back" button.
8.3.4. Otherwise, press "Continue". At this point the tool will process the files, and it may take a few minutes. While the tool is processing the files, it will be not responsive. When the files have been processed, the tool will take you to the "Function Management Page".
8.4. On the "Function Management Page" take the following steps:
8.4.1. To add any function whose model you want to extract, simply select the function from the drop down and press the "Add" button.



8.4.2. If you want to see some statistics related to the function. Take the following steps:
8.4.2.1. Select the function from the drop down and then press the "Inspect" button.
This will open the inspect screen.



8.4.2.2. The inspect screen will tell you the following info about the function:
8.4.2.2.1. The total number of "defined functions" (ref 5.3) along with their names in the selected function.
8.4.2.2.2. The total number of "undefined functions" (ref 5.4) along with their names in the selected function.
8.4.2.2.3. The total number of "Global Variables" along with their names in the selected function.
8.4.2.4. The total number of "Local Variables" along with their names in the selected function.
8.4.2.5. The total number of "Undefined Variables" (ref 5.6) along with their names in the selected function.




8.4.3. A list of all functions that are called in the functions that you have selected is also available.



8.4.4. To remove a selected function:
8.4.4.1. Select the function you want to remove from the drop down.



8.4.4.2. Once selected press the "Remove" button to remove the button from the selection.



8.4.4.3. Alternately, you can press the "RemoveAll" button to remove all the selected functions.




8.4.5. Once you have selected your desired functions to convert to models, select your level of compactness. The level of compactness does not have any functional impact on the generated model, but rather it is more cosmetic. It depends on the user which option they opt.



8.4.6. Write the name you would like to give to the generated xml (UPPAAL model file).



8.4.7. When you want to generate the model, click on the "Finish" button.



8.4.8. Once the finish button has been pressed, the tool will do the following:
8.4.8.1. The tool will generate an xml with the name you specified (ref 8.4.6) in the Directory you specified (ref 8.2.2).
8.4.8.2. The tool will generate log files in the Directory you specified (ref 8.2.1).
8.4.8.3. The tool will take you back to the "Files Management Screen".
